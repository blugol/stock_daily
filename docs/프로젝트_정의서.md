# 프로젝트 정의서 (HTS 연동 복기 메모장)

본 문서는 키움증권 HTS(영웅문4)와 연동하여 동작하는 **오프라인 개인화 주식 복기/매매일지 프로그램**의 개발 목적과 기술적 구현 방식을 정리한 정의서입니다.

---

## 1. 개발 목적 (Purpose)

### 📌 해결하고자 하는 문제 (Pain Points)
1. **HTS 통합메모의 용량 제한**: 영웅문 내장 통합메모는 최대 입력 글자 수 제한이 있어 상세한 기록이 어렵습니다.
2. **가독성 및 분류 부재**: 줄바꿈만 가능한 단순 텍스트 박스 형태라 작성 항목이 분리되지 않고, 결국 주먹구구식 일기장 형태로 전락합니다.
3. **분석 데이터 활용 한계**: 과거 메모를 검색하거나 매수/매도 시점의 조건별로 필터링하여 모아볼 수 있는 기능이 없습니다.

### 💡 얻고자 하는 가치 (Target Value)
* **체계적인 입력 폼**: `[매수이유]`, `[매도이유]`, `[잘한 점]`, `[보완할 점]` 등 명확한 입력 양식을 통해 일관성 있는 복기 데이터를 축적합니다.
* **HTS 종목 연동 편의성 유지**: 기존 통합메모의 가장 큰 장점인 **"HTS에서 종목 클릭 시 해당 종목 메모장 자동 열림"** 동작을 100% 동일하게 재현합니다.
* **보안 및 오프라인 보장**: 개인 자산 및 트레이딩 데이터가 외부로 유출되거나 HTS 보안 모듈에 차단되는 일이 없도록 **인터넷 통신이 전혀 없는 100% 로컬 환경**으로 구축합니다.

---

## 2. 핵심 기능 정의 (Key Features)

### 🔄 1. 실시간 로컬 종목명 감지 (HTS Sync)
* **기능**: 백그라운드에서 실행되는 파이썬 프로그램이 영웅문4의 자식 창 구조(`AfxWnd110` 클래스)를 주기적으로 스캔하여 사용자가 현재 보고 있는 주식 종목명(예: "윤성에프앤씨")을 실시간으로 감지합니다.
* **동작**: 종목명이 감지되는 순간, 메모 프로그램의 UI 화면이 해당 종목의 일지 페이지로 즉시 자동 전환됩니다.

### 📝 2. 구조화된 매매일지 템플릿 (Structured Logging)
* **화면 분할 입력**: 한 칸의 텍스트 박스가 아닌 목적별 입력란 제공
  * `[매매 구분]`: 주간 매매 / 장기 투자 / 관심종목 분석 등
  * `[매수/진입 관점]`: 돌파 / 눌림목 / 뉴스재료 / 수급 등
  * `[매도/청산 관점]`: 목표가 도달 / 지지선 이탈 / 타점 실수 등
  * `[종합 피드백]`: 심리 상태 조절, 당일 매매의 핵심 반성 포인트
* **무제한 기록**: 글자 수 제한 없이 텍스트 기록 가능.

### 📂 3. 오프라인 로컬 저장소 (Local Storage)
* **저장 방식**: 외부 데이터베이스 서버 대신, 프로그램 폴더 내에 `data/윤성에프앤씨.json`과 같이 **텍스트 기반 JSON 파일**로 안전하게 보관합니다.
* **장점**: 백업이 매우 쉽고(폴더만 복사하면 끝), 언제든 일반 메모장으로 열어볼 수 있으며, 인터넷 연결이 없어도 데이터 손실 우려가 전혀 없습니다.

## 3. 기술 아키텍처 및 구현 방식 (Technical Method)

프로그램은 **백그라운드 감지기**와 **사용자 화면(UI)**의 두 부분으로 구성됩니다.

```
┌──────────────┐          1. 창 텍스트 감시          ┌────────────────┐
│ 영웅문4 HTS  ├────────────────────────────────────>│ Python 감지기  │
└──────────────┘                                     └───────┬────────┘
                                                             │ 2. 종목명 전송
┌──────────────┐          3. 자동 로드 및 저장       ┌───────▼────────┐
│ 로컬 데이터  │<────────────────────────────────────┤  로컬 웹 GUI   │
│ (JSON 파일)  │                                     └────────────────┘
└──────────────┘
```

### 🛠️ 기술 스택 (Tech Stack)
* **감지기 (Backend)**: Python + `win32gui` (윈도우 API 연동)
* **사용자 화면 (Frontend/UI)**: 로컬 웹 브라우저를 기반으로 하는 **HTML/CSS/JS (Flask/FastAPI 기반 로컬 웹서버)** 
  * *이유*: 데스크톱 기본 GUI(Tkinter 등)보다 **압도적으로 세련되고 예쁜 다크모드 디자인**을 적용할 수 있어 시각적 몰입감을 높여줍니다. HTS 바로 옆에 띄우기 최적화된 컴팩트한 레이아웃을 제공합니다.
* **데이터 포맷**: `JSON` (파일 기반 데이터베이스)

## 4. 향후 개발 로드맵 (Roadmap)

1. **[1단계 - 완료]** 로컬 HTS 종목명 실시간 감지 검증
2. **[2단계]** 파이썬 백그라운드 감지기와 예쁜 로컬 웹 UI 화면 연동 프로토타입 제작
3. **[3단계]** 구조화된 입력 항목 정의 및 로컬 파일 자동 로드/저장 기능 구현
4. **[4단계]** 최종 사용성 검증 및 단독 더블클릭 실행 환경 구축
