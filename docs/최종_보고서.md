## 1. 프로젝트 개요 및 목적

### 배경
트레이더로서 매일의 매매를 체계적으로 복기하고 분석하는 것은 실력 향상의 핵심이다. 기존에는 키움증권 영웅문4 HTS 내장 메모장에 메모를 남기거나 별도 텍스트 파일로 기록을 관리했으나, 이는 종목별 이력 추적이 어렵고 수익률 계산을 수동으로 해야 하는 불편함이 있었다.

### 목표
- **영웅문4와 실시간 자동 연동**: HTS에서 종목을 바꾸면 대시보드가 자동으로 해당 종목의 복기 화면으로 전환
- **마틴 분할매수 계획의 정량화**: 1-1-2-4 비중 원칙을 계산기로 실시간 추적
- **매매 성과의 자동 결산**: 보유기간·수익률을 자동 계산하여 기록
- **100% 오프라인 구동**: 보안 민감한 트레이딩 PC에서 외부 서버 없이 로컬만으로 동작

## 2. 시스템 아키텍처

```
[영웅문4 HTS] ──Win32 API──▶ [server.py:8000] ──HTTP──▶ [Chrome 브라우저]
                                     │
                              [data/*.json]  ◀──자동저장──  [app.js]
                              [backup/]       ──자동백업──▶
```

### 핵심 설계 원칙
- **Zero 외부 의존**: 인터넷 연결 없음, 외부 DB 없음, 클라우드 없음
- **보안 프리**: Win32 `GetWindowText` 표준 API만 사용 (프로세스 인젝션·API 후킹 없음)
- **단일 명령 실행**: `python server.py` 하나로 전체 시스템 기동

## 3. 구현 기능 전체 목록

### ① 실시간 HTS 종목 연동
- 1초 폴링으로 영웅문4 메인 창 핸들(`HWND`) 탐색
- `AfxWnd110` 클래스 컨트롤에서 종목명 텍스트 자동 추출
- 무효어(뉴스, 소리, 계좌, 잔고 등) 필터링으로 오감지 방지
- 종목 전환 시 기존 데이터 자동 저장 후 신규 종목 데이터 자동 로드

### ② 종목 목록 사이드바
- 전체 / 진행중 / 결산완료 3가지 필터 탭
- 종목명 실시간 검색
- 종목 이름만 표기 (태그 비표시로 깔끔하게)
- `×` 버튼으로 종목 삭제 (즉시 삭제가 아닌 `backup/` 폴더로 이동 — 휴지통 방식)

### ③ 차트 상태 분류 (Trend Status)
- 하락 추세 진행중 / 눌림목 지지중 / 수렴 및 박스권 / 추세 상단 돌파
- 4가지 상태를 색상 강조 라디오 버튼 카드로 즉각 선택
- 선택 즉시 JSON 자동 저장

### ④ 노션식 동적 키워드 태그
- 텍스트 입력 후 `Enter` → 알약형 배지 즉시 생성
- 배지 내 `×` 클릭으로 개별 삭제
- `["부채150%초과", "자사주10%"]` 형태로 JSON 저장

### ⑤ 1-1-2-4 마틴 분할매수 계산기
- 1차 수량 입력 → 2차(1배)·3차(2배)·4차(4배) 목표 수량 자동 연산
- 차수 무제한 동적 추가·삭제 (`분할매수 차수 추가` 버튼)
- 양봉 확인 체크박스 클릭 → 오늘 날짜 자동 기록
- 체크된 차수 기준 **실시간 평단가** 계산·표시
- **진입 비중 분석 결과**: 목표 비중 대비 실제 매수 주수의 초과/부족 즉시 계산
- **다음 진입 예정 비중 추천**: 부족분이 있으면 다음에 채울 주수 권장 표시

### ⑥ 매매 자동 결산 시스템
- 최종 매도 단가 + 최종 매도일 입력 → `[최종 청산]` 버튼으로 결산 실행
- 보유기간(일수)·수익률(%) 자동 계산
- 4칸 그리드 표로 결과 표시
  - **매수 정보**: 1차 진입일 + 평균 매수단가
  - **매도 정보**: 최종 매도일 + 매도단가
  - **보유 기간**: 계산된 일수 (예: `10D`)
  - **최종 수익률**: 소수점 2자리 퍼센트 (예: `+15.84%`)
- 결산 완료 후 `수정` 버튼으로 잠금 해제 → 값 직접 수정 → `완료`로 재저장
- 매도일·단가 변경 시 보유기간·수익률 **실시간 자동 재연산**
- 결산 완료 시 타임라인 로그에 결산 기록 자동 추가

### ⑦ 타임라인 메모 일지
- `Enter` 전송 / `Shift+Enter` 줄바꿈
- `[YYYY-MM-DD HH:MM:SS]` 타임스탬프 자동 부여
- 시간순 블록으로 누적 기록
- 레이아웃 이탈 버그 해결 (내부 독립 스크롤바 구성)

### ⑧ 데이터 유실 방지 자동 백업
- **삭제 시**: `backup/[타임스탬프]_[종목명].json` 으로 이동 (휴지통 방식)
- **수정 시**: 덮어쓰기 전 기존 파일을 `backup/`에 자동 복사
- 실수나 오작동으로 데이터 파괴 불가

### ⑨ 과거 매매기록 일괄 이관
- `매매.txt` 파일(텍스트 형식 과거 기록)을 파싱하여 각 종목의 결산 JSON으로 자동 생성
- `거래내역.csv` (HTS 내보내기 데이터) 파싱 및 이관 지원
- 이관된 종목은 결산완료 상태로 사이드바에 즉시 반영

## 6. 개발 과정에서 해결한 주요 난제

### 문제 1: HTS 보안 프로그램 충돌 우려
- **원인**: 영웅문4는 AhnLab Safe Transaction 보안 솔루션을 사용하며, API 후킹·프로세스 메모리 접근 시 차단·종료될 수 있음
- **해결**: `GetWindowText` 표준 Win32 API만으로 창 텍스트를 수집하는 방식 채택. 보안 솔루션이 감지·차단하지 않는 무해한 접근 방식

### 문제 2: HTS 창에서 종목명과 일반 텍스트 구분
- **원인**: 영웅문4 하위 창이 2,000여 개이며, `AfxWnd110` 클래스에 "뉴스", "조건검색" 등 비종목 텍스트가 혼재
- **해결**: `EXCLUDED_WORDS` 집합 + `POTENTIAL_NAME_PATTERN` 정규식 + 문자 수(2~10자) 3중 필터링

### 문제 3: 서버를 백그라운드 프로세스로 실행 시 HTS 감지 불가
- **원인**: Windows 보안 세션 분리로 인해 백그라운드 서비스 세션에서는 로그인 데스크탑의 창 목록에 접근 불가
- **해결**: 반드시 트레이더님이 직접 CMD에서 `python server.py`를 실행해야 함을 문서화

### 문제 4: 구버전 데이터 파일 로드 시 브라우저 오류
- **원인**: 과거 기록 이관으로 생성된 JSON에 `dynamic_tags`, `sell_date` 등 신규 필드 누락
- **해결**: `loadMemoData()` 내 방어적 스키마 초기화(Defensive Initialization) — 누락 필드를 기본값으로 자동 채움

### 문제 5: 결산 완료 후 수정이 안 되는 문제
- **원인**: 결산 완료 상태에서 UI가 읽기 전용으로 고정되어 날짜·단가 수정 불가
- **해결**: `수정` 버튼으로 편집 모드 진입 → 값 변경 후 `완료` 클릭 시 보유기간·수익률 재연산 및 JSON 저장

### 문제 6: 타임라인 메모창이 화면 아래로 이탈
- **원인**: 우측 패널 내 결산창과 메모창이 고정 높이 없이 내용에 따라 늘어나 레이아웃 붕괴
- **해결**: 타임라인 박스에 `max-height` 제한 + 독립 `overflow-y: auto` 스크롤바 구성
