# 구현 계획서 (HTS 연동 복기 메모장 v2.0)

키움증권 HTS(영웅문4) 종목 감지 기능과 연동되는 **로컬 오프라인 웹 기반 메모장 대시보드** 구현 계획입니다.

## 1. 아키텍처 구조

1. **로컬 서버 (`server.py`)**:
   - 파이썬 기본 라이브러리인 `http.server`를 사용하여 구현 (별도 라이브러리 설치가 필요 없어 안전하며 오프라인에서 즉시 실행 가능).
   - HTS 종목명을 감지하는 백그라운드 스레드를 구동합니다.
   - REST API 제공:
     - `GET /api/status`: 현재 활성화된 종목명을 반환합니다.
     - `GET /api/memo?stock=...`: 특정 종목의 JSON 저장 데이터를 읽어옵니다.
     - `POST /api/memo?stock=...`: 특정 종목의 입력 데이터를 로컬 `data/` 폴더에 파일로 저장합니다.
2. **웹 화면 (`index.html`, `style.css`, `app.js`)**:
   - 눈의 피로를 최소화하는 **택티컬 다크 테마(Dark Theme)** 적용.
   - HTS 감지 상태를 1초 주기로 폴링(Polling)하여 종목 변경 시 자동으로 화면을 전환 및 자동 저장합니다.
   - **노션 스타일의 동적 태그**: 사용자가 글을 쓰고 엔터를 누르면 알약 모양의 태그가 동적으로 생성 및 삭제되는 UI.
   - **1-1-2-4 마틴 분할매수 계산기**: 1차 수량과 단가 입력 시 배수 계산, 양봉 확인 체크 시 매수일 기록 및 현재 총 평단가 실시간 갱신.
   - **최종 청산 결산기**: 매도 단가 입력 후 청산 시 보유일수(D) 및 수익률(%) 자동 연산, 정형화된 텍스트 스탬프 출력.
   - **타임라인 피드**: 트위터 방식의 수시 코멘트 기록 시스템.

## 3. 검증 계획 (Verification)

1. `python server.py`를 실행하여 포트 8000번에서 웹서버 구동 확인.
2. 브라우저에서 `http://localhost:8000` 접속 테스트.
3. 영웅문 HTS에서 종목을 바꿀 때 브라우저 화면의 종목명이 연동되어 바뀌고 과거 작성한 데이터가 실시간으로 로드되는지 확인.
4. 마틴 계산기, 결산기, 태그 입력이 정상 동작하며 작성 내용이 `data/[종목명].json` 파일로 올바르게 저장/복구되는지 확인.
