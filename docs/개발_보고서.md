# 개발 보고서 (HTS 연동 복기 대시보드 v2.0 - 최종 고도화)

본 보고서는 키움증권 HTS(영웅문4)와 연동하여 구동되는 **로컬 오프라인 복기 대시보드 v2.0**의 전체 아키텍처, 데이터 스키마 및 최종 보완·개선된 기능 명세를 상세히 기술합니다.

---

## 1. 시스템 아키텍처 & 보안 모델

프로그램은 성능 최적화와 트레이딩 PC의 보안 안정성을 보장하기 위해 **100% 오프라인 로컬 독립 실행 구조**로 설계되었습니다.

### 🛡️ 보안 및 안전성 강화 설계
* **인터넷 연결 Zero**: 외부 웹 호출, DB 서버 연동, 라이브러리 자동 다운로드 코드가 원천 배제되어 로컬 데이터 보안이 유지됩니다.
* **보안 프로그램 우회 우려 해소**: HTS의 AhnLab Safe Transaction 보안 솔루션이 차단하는 프로세스 인젝션, API 후킹 등의 위험한 방식을 쓰지 않고, 표준 Windows API(`GetWindowText`)를 활용한 단순 창 텍스트 수집 방식을 채택하여 감지 중 차단 위협이 발생하지 않습니다.
* **로컬 캐시 방지 및 서버 최적화**: 웹 서버 통신 시 `Cache-Control` 헤더 및 스크립트 버전 파라미터를 추가하여 브라우저에 구버전 스크립트가 캐싱되어 연동이 멈추는 오작동을 차단했습니다.

## 2. 기능별 상세 명세

### 🔄 ① 실시간 HTS 종목 연동 (HTS Sync)
HTS에서 분석/매매 종목을 변경할 때마다 대시보드가 이를 자동으로 인지하여 해당 종목의 과거 일지를 열어주는 핵심 엔진입니다.

* **동작 원리**: 
  1. 1초마다 시스템 전체에서 `영웅문4` 타이틀을 가진 최상위 윈도우 핸들(`HWND`)을 탐색합니다.
  2. 영웅문 창이 확인되면, 내부의 모든 하위 자식 창 중 종목명이 업데이트되는 `AfxWnd110` 클래스의 컨트롤을 스캔합니다.
  3. 추출된 텍스트 중 무효어(예: 뉴스, 소리, 예상 등)를 필터링하고 종목명 패턴(2~10자 한글/영문)에 맞는 유효한 종목명(예: `윤성에프앤씨`)을 실시간 감지합니다.
* **화면 연동**: 종목명 변경이 감지되면 프론트엔드 UI가 자동으로 기존 데이터를 로컬 파일에 저장(Auto Save)하고, 새로운 종목의 복기 데이터(`data/[종목명].json`)를 로드하여 화면을 갱신합니다.

### 🏷️ ② 차트 상태 분류 및 노션식 동적 태그
작성하는 복기 메모에 메타데이터를 부여하여 훗날 필터링 및 조건별 검색을 용이하게 만드는 기능입니다.

* **추세 상태 (Trend Status)**: 
  * 트레이더가 현재 직면한 차트 자리를 강제 분류하도록 드롭다운 대신 직관적인 **라디오 버튼 카드** 형태로 상단 배치했습니다.
  * 상태별 고유 색상 강조: 하락 추세(블루), 눌림목 지지(퍼플), 수렴/박스권(티알/그린), 상단 돌파(레드).
* **동적 키워드 태그 (Notion-style Tags)**:
  * 미리 정의된 체크박스가 아닌, 사용자가 텍스트 입력 후 `Enter` 키를 입력하면 즉시 알약 형태의 **태그 배지(Badge)**가 생성되는 구조입니다.
  * 배지 내부의 `x` 버튼을 누르면 즉시 삭제됩니다.
  * 데이터는 `["부채150%초과", "자사주10%"]` 형태의 문자열 배열로 JSON에 실시간 저장됩니다.

### 📊 ③ 1-1-2-4 마틴 분할매수 자동 계산기 (차수 확장 및 비중 분석 탑재)
트레이더가 기계적이고 통제된 분할매수 원칙을 지킬 수 있도록 지원하며, 매수 비중을 계산해 주는 실전 트레이딩 계산 엔진입니다.

* **차수 무제한 확장 및 삭제 기능**:
  * 기존의 고정된 4차 분할매수 한계를 탈피하고, **`분할매수 차수 추가`** 버튼을 통해 5차, 6차 등 동적으로 추가할 수 있도록 개선했습니다.
  * 실수로 누른 차수를 바로 제거할 수 있는 개별 **`삭제(&times;)`** 버튼을 행별로 지원합니다.
* **양봉 매수 실행 트리거 (Execution)**:
  * 각 회차별로 **`[ ] 양봉 확인`** 체크박스가 제공됩니다.
  * 이 체크박스를 클릭(매수 집행)하는 즉시 **오늘 날짜(YYYY-MM-DD)**가 해당 회차의 매수일로 자동 고정되며 기록됩니다. 체크 해제 시 초기화됩니다.
* **현재 매수 평단가 실시간 계산**:
  * 체크박스가 활성화된 회차들의 `(수량 * 단가)` 누적합을 `총 보유 수량`으로 나누어 **현재 시점의 정확한 가중평균단가(평단가)**를 실시간으로 연산하여 대시보드 중앙에 큰 강조 텍스트로 표기합니다.
* **진입 비중 분석 결과 요약 (Weight Analysis)**:
  * 마틴 계산기 최하단에 **`진입 비중 분석 결과`** 요약판을 단일로 구성하여 가독성을 높였습니다.
  * 매 진입 시마다 설정한 1차 수량 기준 마틴 비율(1:1:2:4 등) 대비 실제 매수한 주식 수가 초과 또는 부족한지 실시간 비교 연산합니다. (예: `2차 1주 초과, 3차 1주 부족, 4차 3주 부족`)
  * **다음 진입 예정 비중 추천**: 비중이 부족할 경우, 다음에 계획대로 채워야 할 회차와 주식 수를 강조합니다. (예: `4차 3주 매수 권장`). 채울 비중이 없으면 `없음 (계획대로 매수 완료)`로 표시됩니다.

### 🏆 ④ 매매 자동 결산 시스템 (실시간 재연산 및 그리드 표 표기)
매매가 완전히 종료(청산)되었을 때 성과를 정량적으로 계산하여 직관적으로 확인하고, 필요시 자유롭게 수정 가능한 시스템입니다.

* **최종 매도일 선택 기능 추가**:
  * 결산 카드에 `최종 매도일` 날짜 피커(Date Picker)를 추가하여 당일 청산뿐만 아니라 과거 청산 기록도 원하는 날짜로 계산할 수 있게 보완했습니다.
* **실시간 상호 작용 재연산**:
  * 결산 완료 상태에서 `최종 매도 단가` 혹은 `최종 매도일`을 변경하면, **보유 기간(D)과 최종 수익률(%)이 실시간으로 자동 재계산**되어 즉시 화면에 반영됩니다.
* **4칸 격자형(Grid) 결과 요약**:
  * 암호 같던 기존 한 줄 텍스트 스탬프와 복사 버튼을 UI에서 과감히 지우고, 깔끔한 4개 카드 그리드로 정보를 표기합니다.
    * **매수 정보 (일자/평단)**: 1차 진입일과 계산된 평균 매수단가 (예: `06-11 (₩ 5,240)`)
    * **매도 정보 (일자/단가)**: 최종 매도일과 매도단가 (예: `06-20 (₩ 6,070)`)
    * **보유 기간**: 계산된 보유일 수 (수동 수정 지원, 예: `10D`)
    * **최종 수익률**: 최종 청산 수익률 (수동 수정 지원, 예: `+15.84%`)
* **잠금식 수정 토글 및 백그라운드 갱신**:
  * 결산 완료창은 기본적으로 읽기 전용으로 잠겨 실수를 막으며, **`수정`** 버튼 클릭 시에만 `보유 기간`과 `최종 수익률`을 강제 기입할 수 있습니다.
  * 수동 수정 후 **`완료`** 버튼을 누르면 내부 백그라운드에서 legacy 스탬프 문자열을 자동 재생성하여 타임라인 로그의 텍스트와 JSON 데이터를 동시 갱신합니다.

### 💬 ⑤ 타임라인형 메모 기록 (Daily Log - 레이아웃 안정화)
매매 도중 수시로 느끼는 뇌동매매의 유혹, 시장 흐름의 변화, 차트의 실시간 관점 등을 타임라인 블록으로 누적 보관하는 일지 시스템입니다.

* **스크롤 깨짐 현상 해결**:
  * 메모 입력 내용이 길어지거나 결산 기록이 다수 추가될 때 우측 패널 레이아웃이 꼬여서 화면 아래로 이탈하던 버그를 해결했습니다.
  * 우측 스크롤 흐름에 종속되도록 레이아웃을 단순화하고, 타임라인 박스 내부에 최대 높이 250px 한도의 단독 스크롤바를 구성하여 가독성을 대폭 향상했습니다.

### 🚨 ⑥ 데이터 유실 방지 자동 백업 시스템 (Recycle Bin & History)
트레이더가 실수로 종목을 지우거나, 데이터를 오작동으로 덮어썼을 때 소중한 데이터 자산이 파괴되는 것을 원천 방지하는 2단계 안전 장치입니다.

* **삭제 시 (휴지통 기능)**:
  * 사이드바 종목의 `&times;` (삭제) 버튼을 누르면 파일이 영구 삭제되는 대신, [backup](file:///c:/Users/user/Desktop/복기/backup) 폴더로 날짜·시간 스탬프가 접두어로 붙은 상태로 안전하게 이동(Move) 처리됩니다.
* **수정 시 (이력 저장 기능)**:
  * 분할매수 정보를 수정하거나 결산 완료 내용을 수정하여 데이터가 덮어써질 때마다, **기존 원본 JSON 파일이 백업 폴더로 자동 복사(Copy)되어 타임스탬프와 함께 백업본으로 영구 보관**됩니다.

## 4. UI/UX 디자인 가이드라인

HTS를 구동 중인 멀티 모니터 환경에서 사용하기에 가장 적합한 **Tactical 다크 모드 인터페이스**로 설계되었습니다.

1. **상단 영역**: 영웅문에서 연동된 대형 종목명 뱃지가 깜빡이는 연동 표시(Pulse Icon)와 함께 중앙에 띄워집니다. 아래에 Notion 식의 태그와 큼직한 추세 선택 라벨이 노출되어 종목 분석 정보를 한눈에 인식할 수 있습니다.
2. **중단 영역 (핵심)**:
   * **좌측**: 마틴 분할매수 계산기 표가 배치되어, 매수 단가를 조정하고 양봉 확인 체크박스를 즉각 누를 수 있습니다. 계산기 하단에는 연둣빛으로 빛나는 총 평단가가 크게 표시됩니다.
   * **우측**: 목표가와 손절가를 기입하여 심리적 손절 기준을 강제합니다. 그 아래에는 매도 완료 시 청산 결과를 계산하는 결산기와 깔끔한 4칸 결산정보 그리드가 배치되어 있습니다.
3. **하단 영역**: 타임라인 일지창이 넓게 배치되어, 장중 떠오르는 아이디어를 즉각 타이핑하여 타임스탬프와 함께 피드로 기록할 수 있습니다.
