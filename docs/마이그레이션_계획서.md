# 마이그레이션 구현 계획서 (HTS 통합메모 -> v2.0 JSON)

기존 HTS 통합메모 텍스트 데이터를 v2.0 대시보드 규격의 JSON 파일들로 자동 변환하는 마이그레이션 계획입니다.

## 1. 마이그레이션 규칙

1. **대상 파일**: `통합메모(과거데이터).txt` (CP949 인코딩) -> `data/[종목명].json` (UTF-8 인코딩)
2. **파싱 엔진**:
   - **종목 분리**: `^\[(\d{6})\]` 패턴을 감지하여 각 종목 블록을 생성합니다.
   - **종목명 및 코드**: 대괄호 안은 `종목코드`, 대괄호 이후 첫 필드/날짜 전까지는 `종목명`으로 자동 분리합니다.
   - **재무 항목**: `재무 :` 뒷부분을 정규식 `[가-힣A-Za-z]+?\d+%?|[가-힣A-Za-z]+`로 파싱하여 노션식 동적 태그(`dynamic_tags`) 배열로 채웁니다.
   - **매수 항목**: `매수 :` 뒷부분의 쉼표로 구분된 가격들을 순서대로 마틴 계산기 1차, 2차 단가에 넣고 체결 체크 처리합니다. 날짜 괄호(예: `(0615)`)가 있으면 `2026-06-15`로 날짜를 파싱합니다.
   - **목표/손절**: 각각 `target_price`, `stop_price`에 숫자로 채웁니다.
   - **실현 수익률**: `[실현 : ...]` 패턴이 있으면 결산 정보(`settlement.yield_rate`)에 넣고 결산 완료 처리합니다.
   - **타임라인 로그**: `YYYY/MM/DD` 또는 `MMDD:` 패턴을 스캔하여 날짜 형식(`YYYY-MM-DD 00:00:00`)으로 표준화하고 내림차순(최신순) 정렬하여 저장합니다.
